# DfM-Ninja

[English version (README.md)](../README.md)

DfM-Ninja は、Dynamics 365 (DfM) のカスタマーサポートエンジニア向けに設計されたフロントエンド専用 SPA (Single Page Application) です。ケース管理、メールテンプレート作成、メタデータ処理、そして Dynamics 365 と LLM 連携ワークフローを橋渡しする高速なインターフェースを提供します。

## 概要

本アプリケーションは **React 19** と **Vite 7** を使用して構築されており、スタイリングには **Tailwind CSS 4** を採用しています。ブラウザ内のみで動作し、ケースデータは **IndexedDB**、設定・テンプレート・カスタム定型文は `localStorage` に保存されるため、バックエンドサーバーは不要です。ケースデータは一括バックアップや JSON 形式での個別エクスポートが可能です。

専用のブックマークレットを介して Dynamics 365 ページと連携し、ケース情報の抽出や Dynamics 環境と Ninja SPA 間のリアルタイムな通信を実現します。

## 主な機能

### 🔍 全文検索 (`Cmd+P` / `Ctrl+P`)
IndexedDB に保存されたすべてのケースを正規表現で検索できます。結果はインタラクティブなアコーディオン形式で表示され、クリックすることで該当するケースやステージへ直接遷移できます。

### 🥷 ブックマークレット連携
ブックマークレットを実行すると Dynamics 365 ページに忍者ボタンが注入されます。これをクリックするとケースメタデータが抽出され、自動的に DfM-Ninja タブにフォーカス（または新規作成）されるため、タブ切り替えの手間が省けます。

### 🛠️ テンプレート作業 UI
- **2カラムレイアウト**: 左側にプロンプト/ガイドライン、右側に入力フィールドを配置した標準化されたワークフロー。
- **オートセーブ**: 入力した内容はリアルタイムで IndexedDB に保存されるため、ナビゲーション中のデータ紛失を防ぎます。
- **動的コンテンツ**: EJS (Embedded JavaScript) をサポートしており、複雑な条件分岐やフォーマットをテンプレート内で実現可能です。
- **カスタム定型文**: LLM プロンプトやメール本文などのテンプレート初期テキストを、テンプレート ID ごとに `localStorage` に保存・管理できます。テンプレート再取込時も既存データとマージされます。
- **リセット動作**: テンプレート内の「リセット」ボタンは、その時点で読み込まれている定型文ソースに戻しつつ、ケースの Step 状態とも同期されます。

### 🚀 アップデート通知機能
ユーザーの状態に合わせて以下の通知を自動で表示します：
- **Welcome**: 新規ユーザー向けのオンボーディング。
- **Upgrade**: 以前のバージョンから移行したユーザー向けの案内。
- **Update**: 新機能リリース時の通知 (v0.6.3+)。

### 🔰 変数一覧モーダル
テンプレートで使用可能なすべての変数を参照・ビルドできます：
- **Editable Var.**: ユーザー定義のグローバル変数。
- **Dynamic Var.**: 日付や NC（次回連絡日）の計算・プレビュー。
- **Stage Var.**: 現在の UI から抽出された変数。
- **Case Var.**: `caseNum`、`caseTitle`、`SLA`、`severity`、各種日時などのケースメタデータ。
- **Sys Temp.**: `attentions` や `teamsDisclaimer` などの登録済みシステムスニペット。

### 💾 インポート / エクスポート
- **Cases**: ケースを ZIP で一括エクスポート / インポート可能。
- **Templates**: `templates.zip` を取り込み、Stage Templates と Sys Temp. を一括更新。
- **Settings**: YAML 設定とカスタム定型文をまとめてエクスポート / インポート可能。
- **レガシーマイグレーション**: 旧 localStorage ベースのケースデータは、起動時に ZIP バックアップを作成した上で IndexedDB へ移行されます。

## はじめかた

1. リポジトリをクローンします。
2. 依存関係をインストールします：
   ```bash
   npm install
   ```
3. 開発サーバーを起動します：
   ```bash
   npm run dev
   ```
4. ブラウザで `http://localhost:5178` にアクセスします。

主な補助コマンド：

- `npm run build` - GitHub Pages 向けビルド（`/DfM-Ninja/` ベース）
- `npm run build:local` - オフライン配布用の `dist.zip` を生成
- `npm run build:templates:bundle` - `templates/` から `templates.zip` を再生成
- `npm run build:bookmarklet` - Bookmarklet ソースを手動ビルド
- `npm run release` - リリース作成

> [!NOTE]
> `src/utils/bookmarkletCode.js` は自動生成ファイルです。`npm run dev` または `npm run build` を実行すると自動的に作成されます。

## ローカル（オフライン）環境での使用

ネットワークのない環境や `file:///` から起動したい場合は、以下のコマンドでパッケージを生成できます：
```bash
npm run build:local
```
これにより、相対パス指定とローカル依存関係（jQuery, EJS）を含んだ `dist.zip` が生成されます。
